= idaptik-game
:toc:

The IDApixiTIK game itself: an asymmetric co-op stealth puzzle-platformer
written in AffineScript, compiling to typed-wasm WasmGC, rendered with
PixiJS through a thin FFI binding layer.

== Status

The current `hyperpolymath/idaptik` repo's `src/` directory (508 .res files)
ports here. The port is gated by Recovery PRs 1-3 (see
`idaptik-developers/docs/RECOVERY.adoc`): the game must build and run on
the existing ReScript stack before the AffineScript port begins.

== Dependencies

* `idaptik-contracts` — Role, SecurityLevel, DeviceKind, Zone, Instruction
* `idaptik-shared` — façade utilities
* `idaptik-dlc-vm` — VM as DLC (loaded at puzzle screens)
* PixiJS 8.8.1 (via FFI in `src/bindings/`)

== Faces

The port uses AffineScript's faces system deliberately:

* *Engine, screens, devices*: canonical AffineScript face. Effect-explicit;
  the typed `VmState` / `Render` / `IO` / `Network` boundaries are
  reviewable.
* *UI rendering glue*: JaffaScript face. JS-shaped syntax over the same
  affine/QTT semantic core. Familiar to anyone arriving from the Pixi
  ecosystem; same guarantees apply.
* *Narrative scripts*: PseudoScript face. Designer-readable.

== UMS↔game integration

The game's existing AssetPack manifest loader is the bridge. UMS produces
generated content as `dlc-manifest.json` artifacts in the same shape;
the game loads them with no game-side code changes. See Recovery PR 4.
